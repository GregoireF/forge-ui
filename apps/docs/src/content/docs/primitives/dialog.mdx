A **Dialog** renders a modal panel inside a portal, with focus trapping, scroll locking, and pointer-outside detection. It follows the [WAI-ARIA Dialog pattern](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/).

## Usage

## Exit animations with forceMount

Pass `forceMount` to `Dialog.Portal` to keep the content in the DOM when closed. The `data-state` attribute becomes `"closed"` instead of unmounting.

```tsx
<Dialog.Portal forceMount>
  <Dialog.Overlay />
  <Dialog.Content>...</Dialog.Content>
</Dialog.Portal>
```

```css
[data-forge-scope="dialog"][data-forge-part="content"][data-state="closed"] {
  animation: fadeOut 0.2s ease forwards;
}

@keyframes fadeOut {
  to { opacity: 0; transform: scale(0.95); }
}
```

## API Reference

### Dialog.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Initial open state (uncontrolled) |
| `onOpenChange` | `(open: boolean) => void` | — | Fires when open state changes |
| `modal` | `boolean` | `true` | Enables focus trap and scroll lock |
| `trapFocus` | `boolean` | `true` | Enable/disable focus trapping |
| `preventScroll` | `boolean` | `true` | Enable/disable scroll lock |
| `hideOthers` | `boolean` | `true` | Hide background from assistive tech |
| `initialFocusEl` | `() => HTMLElement \| null` | — | Element to focus on open |
| `finalFocusEl` | `() => HTMLElement \| null` | — | Element to focus on close |
| `onOpenAutoFocus` | `(e: Event) => void` | — | Before first element is focused on open |
| `onCloseAutoFocus` | `(e: Event) => void` | — | Before focus is restored on close |
| `onEscapeKeyDown` | `(e: KeyboardEvent) => void` | — | Escape key inside dialog |
| `onPointerDownOutside` | `(e: PointerEvent) => void` | — | Pointer down outside content |
| `onInteractOutside` | `(e: PointerEvent \| FocusEvent) => void` | — | Any interaction outside |

### Dialog.Portal

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `container` | `HTMLElement \| null` | `document.body` | Custom portal container |
| `forceMount` | `boolean` | `false` | Keep children in DOM when closed |

### Dialog.Trigger / Dialog.Close

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Merge props onto child element |

### Dialog.Content / Dialog.Overlay

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Merge props onto child element |
| `forceMount` | `boolean` | `false` | Keep in DOM when closed |

Content also accepts per-instance overrides of all interaction callbacks from `Dialog.Root`.

## Data attributes

| Attribute | Element | Values |
|-----------|---------|--------|
| `data-forge-scope` | All parts | `"dialog"` |
| `data-forge-part` | All parts | `"trigger"` \| `"overlay"` \| `"content"` \| `"title"` \| `"description"` \| `"close"` |
| `data-state` | `overlay`, `content` | `"open"` \| `"closed"` |

## Accessibility

- `Dialog.Content` renders with `role="dialog"` and `aria-modal="true"`
- `Dialog.Title` is wired to `aria-labelledby` on content
- `Dialog.Description` is wired to `aria-describedby` on content
- Focus is trapped inside the dialog while open
- `Escape` closes the dialog
- Focus returns to the trigger on close
- Background content is hidden from screen readers via `aria-hidden`

<Aside type="caution">
  `Dialog.Title` is required for accessibility. A dev-mode warning will fire if the dialog opens without a title or an `aria-label`.
</Aside>

## Keyboard interactions

| Key | Action |
|-----|--------|
| `Enter` / `Space` | Opens dialog when trigger is focused |
| `Escape` | Closes the dialog |
| `Tab` | Cycle focus within the dialog |
| `Shift + Tab` | Reverse-cycle focus within the dialog |
