A **Radio Group** presents a set of mutually exclusive choices. Only one item can be checked at a time. Navigation uses roving tabIndex so the group behaves as a single tab stop.

## Usage

<Tabs>
  <TabItem label="React">
    ```tsx
    import { RadioGroup } from "@forge-ui/react";

    export function PlanSelector() {
      return (
        <RadioGroup.Root defaultValue="pro" name="plan">
          {["free", "pro", "enterprise"].map((plan) => (
            <RadioGroup.Item key={plan} value={plan}>
              <RadioGroup.Radio />
              <RadioGroup.Label>{plan}</RadioGroup.Label>
              <RadioGroup.HiddenInput />
            </RadioGroup.Item>
          ))}
        </RadioGroup.Root>
      );
    }
    ```
  </TabItem>
  <TabItem label="Vue">
    ```vue
    <script setup lang="ts">
    import {

    } from "@forge-ui/vue";

        <RadioGroupItem v-for="plan in plans" :key="plan" :value="plan">

          <RadioGroupLabel>{{ plan }}</RadioGroupLabel>
          <RadioGroupHiddenInput />
        </RadioGroupItem>
      </RadioGroupRoot>
    </template>
    ```

## API Reference

### RadioGroup.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | ã | Controlled selected value |
| `defaultValue` | `string` | ã | Uncontrolled initial value |
| `disabled` | `boolean` | `false` | Disable the entire group |
| `required` | `boolean` | `false` | Mark as required |
| `name` | `string` | ã | Form field name (renders hidden inputs) |
| `orientation` | `"horizontal" \| "vertical"` | `"vertical"` | Affects arrow key navigation |
| `onValueChange` | `(value: string) => void` | ã | Fires on selection change |

### RadioGroup.Item

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | **required** | The value this item represents |
| `disabled` | `boolean` | `false` | Disable this item individually |

### RadioGroup.Radio / RadioGroup.Label / RadioGroup.HiddenInput

These parts inherit their value and checked state from the parent `Item` context. No additional props required.

## Accessibility

- Group has `role="radiogroup"` and `aria-required`
- Each radio has `role="radio"`, `aria-checked`, and uses roving `tabIndex` (only the selected item is in the tab sequence)
- Navigation selects AND focuses simultaneously ã matches the [WAI-ARIA Radio Group pattern](https://www.w3.org/WAI/ARIA/apg/patterns/radio/)

## Data attributes

| Attribute | Element | Values |
|-----------|---------|--------|
| `data-forge-scope` | All parts | `"radio-group"` |
| `data-forge-part` | All parts | `"root"` \| `"item"` \| `"radio"` \| `"label"` |
| `data-state` | `item`, `radio` | `"checked"` \| `"unchecked"` |
| `data-disabled` | All parts | `""` |

## Keyboard interactions

| Key | Action |
|-----|--------|
| `ArrowDown` / `ArrowRight` | Select next item |
| `ArrowUp` / `ArrowLeft` | Select previous item |
| `Space` | Select focused item |
