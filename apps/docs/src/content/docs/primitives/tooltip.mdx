A **Tooltip** shows a short label when the trigger is hovered or focused. It is purely informational and should not contain interactive content.

## Tooltip.Provider

Wrap your application (or section) in `Tooltip.Provider` to share delay configuration across multiple tooltips. This prevents a staggered "popcorn" effect when moving between adjacent triggers.

```tsx
<Tooltip.Provider delayDuration={400} skipDelayDuration={200}>
  {/* multiple Tooltip.Root instances */}
</Tooltip.Provider>
```

## API Reference

### Tooltip.Provider

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `delayDuration` | `number` | `700` | Ms before tooltip opens |
| `skipDelayDuration` | `number` | `300` | Ms within which a second tooltip opens instantly |
| `disableHoverableContent` | `boolean` | `false` | Prevent hovering into the tooltip content |

### Tooltip.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Initial open state |
| `onOpenChange` | `(open: boolean) => void` | — | Fires on state change |
| `delayDuration` | `number` | — | Override Provider delay per tooltip |
| `placement` | `Placement` | `"top"` | Floating placement |

### Tooltip.Anchor

An optional element to use as the positioning reference instead of the trigger. Useful when the visual anchor differs from the interactive trigger:

```tsx
<Tooltip.Root>
  <Tooltip.Trigger>Hover me</Tooltip.Trigger>
  <Tooltip.Anchor asChild>
    <div className="anchor-target" />
  </Tooltip.Anchor>
  <Tooltip.Portal>
    <Tooltip.Content>Positioned relative to the anchor div</Tooltip.Content>
  </Tooltip.Portal>
</Tooltip.Root>
```

### Tooltip.Arrow

A renderless compound that merges arrow positioning props onto its child element:

```tsx
<Tooltip.Content>
  This is a tooltip
  <Tooltip.Arrow>
    <svg width="10" height="5" viewBox="0 0 30 10">
      <polygon points="0,0 30,0 15,10" />
    </svg>
  </Tooltip.Arrow>
</Tooltip.Content>
```

The child receives `data-forge-part="arrow"` and inline `style` for absolute positioning. Use `data-forge-part="arrow-tip"` on a child element for the tip point.

### Tooltip.Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `forceMount` | `boolean` | `false` | Keep in DOM when closed (for exit animations) |

## Data attributes

| Attribute | Element | Values |
|-----------|---------|--------|
| `data-forge-scope` | All parts | `"tooltip"` |
| `data-forge-part` | All parts | `"trigger"` \| `"anchor"` \| `"positioner"` \| `"content"` \| `"arrow"` \| `"arrow-tip"` |
| `data-state` | `content` | `"open"` \| `"closed"` |
| `data-side` | `content` | `"top"` \| `"bottom"` \| `"left"` \| `"right"` |

## CSS variables

| Variable | Description |
|----------|-------------|
| `--forge-revealed` | `1` when positioned and ready to display |

## Accessibility

- Tooltip content has `role="tooltip"` and is wired to the trigger via `aria-describedby`
- The trigger is keyboard focusable — tooltip appears on focus, closes on blur
- Tooltip content is not interactive (links/buttons inside tooltips violate WCAG)

<Aside type="tip">
  Use `Tooltip.Provider` at the app root to share delay settings across all tooltips. Without it, each `Tooltip.Root` manages its own delay independently.
</Aside>
