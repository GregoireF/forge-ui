A **Hover Card** is a non-modal floating panel with a delay. Unlike a Tooltip, it can contain interactive content (buttons, links). Intended for "preview" UX ã e.g. hovering a user mention to see their profile.

## Delays

```tsx
<HoverCard.Root openDelay={300} closeDelay={150}>
  ...
</HoverCard.Root>
```

The `closeDelay` gives users time to move their cursor from the trigger into the card content without it closing.

## Exit animations with forceMount

```tsx
<HoverCard.Portal forceMount>
  <HoverCard.Content>...</HoverCard.Content>
</HoverCard.Portal>
```

```css
[data-forge-scope="hover-card"][data-forge-part="content"][data-state="closed"] {
  animation: fadeOut 0.15s ease forwards;
}
```

## API Reference

### HoverCard.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | ã | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Initial open state |
| `onOpenChange` | `(open: boolean) => void` | ã | Fires on state change |
| `openDelay` | `number` | `700` | Ms before opening |
| `closeDelay` | `number` | `300` | Ms before closing |
| `placement` | `Placement` | `"bottom"` | Floating placement |

### HoverCard.Portal

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `forceMount` | `boolean` | `false` | Keep content in DOM when closed |

### HoverCard.Trigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Merge props onto child element |

### HoverCard.Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Merge props onto child element |
| `forceMount` | `boolean` | `false` | Keep in DOM when closed ã required for exit animations |

### HoverCard.Arrow

Renderless ã merges `getArrowProps()` onto its child element. Typically a `<svg>` with a triangle path.

```tsx
<HoverCard.Arrow>
  <svg width="10" height="5" viewBox="0 0 10 5">
    <path d="M0 0L5 5L10 0Z" fill="white" />
  </svg>
</HoverCard.Arrow>
```

## Data attributes

| Attribute | Element | Values |
|-----------|---------|--------|
| `data-forge-scope` | All parts | `"hover-card"` |
| `data-forge-part` | All parts | `"trigger"` \| `"positioner"` \| `"content"` \| `"arrow"` |
| `data-state` | `content` | `"open"` \| `"closed"` |
| `data-side` | `content` | `"top"` \| `"bottom"` \| `"left"` \| `"right"` |

## CSS variables

| Variable | Description |
|----------|-------------|
| `--forge-revealed` | `1` when positioned and ready to display |

## Accessibility

- `HoverCard.Content` renders with `role="dialog"` and `aria-haspopup="dialog"` on the trigger
- Unlike Tooltip, content may contain interactive elements
- The card stays open while the cursor is inside the content (grace period for movement)
