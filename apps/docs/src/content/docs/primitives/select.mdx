A **Select** renders a button trigger that opens a floating listbox of options. It follows the [WAI-ARIA Listbox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/).

## Placeholder

Two approaches for placeholder content:

**Simple text** ã pass the `placeholder` prop on `Select.Value`:
```tsx
<Select.Value placeholder="Pick a fruitãÎ" />
```

**Rich content or distinct styling** ã use `Select.Placeholder` for CSS targeting and JSX flexibility:
```tsx
<Select.Trigger>
  <Select.Value>
    <Select.Placeholder>
      <span style={{ opacity: 0.5 }}>Pick a fruitãÎ</span>
    </Select.Placeholder>
  </Select.Value>
</Select.Trigger>
```

`Select.Placeholder` renders only when no value is selected and gets `data-forge-part="placeholder"` for styling.

## API Reference

### Select.Root

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | ã | Controlled selected value |
| `defaultValue` | `string` | ã | Uncontrolled initial value |
| `onValueChange` | `(value: string) => void` | ã | Fires when value changes |
| `open` | `boolean` | ã | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | ã | Fires when open changes |
| `disabled` | `boolean` | `false` | Disable the select |
| `placement` | `Placement` | `"bottom-start"` | Floating placement |
| `placeholder` | `string` | ã | Default placeholder for `Select.Value` |

### Select.Value

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `placeholder` | `string` | ã | Text shown when no value is selected |
| `children` | `ReactNode` | ã | Custom placeholder slot (shown when no value) |

### Select.Item

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | **required** | Option value |
| `disabled` | `boolean` | `false` | Disable this option |
| `asChild` | `boolean` | `false` | Merge props onto child |

### Select.Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `forceMount` | `boolean` | `false` | Keep in DOM when closed (for exit animations) |

### Select.Portal

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `container` | `HTMLElement` | `document.body` | Portal target element |
| `forceMount` | `boolean` | `false` | Keep portal when closed |

### Select.Placeholder

Renders only when no value is selected. Accepts standard HTML span attributes.

## Data attributes

| Attribute | Element | Values |
|-----------|---------|--------|
| `data-forge-scope` | All parts | `"select"` |
| `data-forge-part` | All parts | `"trigger"` \| `"value"` \| `"placeholder"` \| `"indicator"` \| `"positioner"` \| `"content"` \| `"option"` \| `"option-text"` \| `"group"` \| `"group-label"` \| `"separator"` |
| `data-state` | `trigger`, `content` | `"open"` \| `"closed"` |
| `data-placeholder` | `trigger` | `""` when no value is selected |
| `data-highlighted` | `option` | `"true"` when keyboard-focused |
| `data-selected` | `option` | `"true"` when chosen |
| `data-disabled` | `option`, `trigger` | `"true"` |

## CSS variables

| Variable | Description |
|----------|-------------|
| `--forge-select-content-transform-origin` | Origin for scale animation on content |
| `--forge-revealed` | `1` when positioned and ready |

## Accessibility

- Trigger has `role="combobox"` and `aria-haspopup="listbox"`, `aria-expanded`
- Content has `role="listbox"`; each item has `role="option"` and `aria-selected`
- Value is synced to a hidden `<select>` for form submission
- Follows the [WAI-ARIA Combobox / Listbox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/)
